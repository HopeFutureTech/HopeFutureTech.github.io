1 简述大模型 Agent 与传统 AI 系统的核心区别
维度传统 AI 系统大模型 Agent
决策能力基于规则或简单ML，决策固定基于LLM推理，可动态决策
工具使用功能预定义，无法扩展可动态调用API，自主规划
上下文理解局限在特定领域跨领域通用理解
自主性被动响应，需人工干预自主规划、执行、反思
学习方式需重新训练/微调提示工程、RAG快速适配
核心区别总结：传统AI是"工具"，大模型Agent是"数字员工"——能理解目标、自主规划、调用工具、持续迭代直到完成任务。

3 Agent 如何在任务执行中做出决策

Agent 决策通常采用 ReAct（Reasoning + Acting） 模式：

Thought → Action → Observation → Thought → Action → ... → Final Answer

决策流程：

1️⃣ 推理（Reasoning）：分析当前状态，评估下一步做什么

2️⃣ 行动（Action）：选择并执行具体动作（调用工具、搜索等）

3️⃣ 观察（Observation）：获取行动结果

4️⃣ 迭代：基于新信息重新推理，直到达成目标

常见决策模式：ReAct（推理行动交替）、CoT（思维链）、ToT（树状思维）、Reflexion（自我反思）

13 AutoGPT 的工作原理

核心循环：

目标设定 → 任务分解 → 循环执行 → 完成判断 循环内： 1. Think（思考）→ 制定计划 2. Act（行动）→ 执行操作 3. Observe（观察）→ 获取结果 4. Memory Update（记忆更新）→ 存储经验 5. 循环直到完成

关键特性：自主决策、长期记忆（向量数据库）、文件系统操作、网页浏览

15 CrewAI 的多 Agent 协作模式

核心概念：

• Agent - 具有特定角色、目标和背景

• Task - 分配给Agent的具体工作

• Crew - 多个Agent的协作组

• Process - 任务执行顺序（顺序/并行/层级）

researcher = Agent(role='研究员', goal='收集信息') writer = Agent(role='撰稿人', goal='撰写报告') task1 = Task(description='研究AI市场', agent=researcher) task2 = Task(description='撰写报告', agent=writer, context=[task1]) crew = Crew(agents=[researcher, writer], tasks=[task1, task2]) result = crew.kickoff()

17 LangGraph 的优势

• 图结构（节点+边）vs 线性链式

• 支持循环、条件分支、并行

• 内置状态管理（StateGraph）

18 什么是 Agent 的 ReAct 模式

ReAct = Reasoning（推理）+ Acting（行动）

让LLM交替进行思考和行动，利用外部工具获取信息

Prompt模板： 问题：{question} 你可以使用以下工具：{tools} 请按以下格式回答： Thought: 你的思考过程 Action: 工具名称[参数] Observation: 工具返回结果 ...（重复Thought/Action/Observation） Final Answer: 最终答案

优点：可解释性强、能处理需要外部信息的复杂问题

19 如何评估 Agent 框架的可扩展性

评估维度：

• 工具扩展 - 是否容易添加新Tool？

• 模型兼容 - 是否支持多种LLM？

• 数据集成 - 支持哪些数据源？

• 社区生态 - 文档完善度、社区活跃度

• 企业特性 - 权限管理、审计日志

框架扩展性特点
LangChain生态最丰富，扩展性最强
LlamaIndex专注RAG，数据层扩展性好
CrewAI多Agent编排，角色系统易扩展
